Open questions

Three architectural decisions blocking TRR. Each needs a decision before the gate can be called.

1
What is OP7?
ICD-09 references a flow called OP7, but it's not in the 18-flow register. Until OP7 is defined — or ICD-09 is reclassified as an internal event — the Capella model can't be used as a TRR verification source.
Blocks: Capella model, TRR gate
Options: Add OP7 as a minimal spec (pool operator audit trail?) or reclassify ICD-09
2
Assessment Org: AOM vs ARM naming mismatch
AOM v4 calls the assessment actor "Assessment Org". ARM WP3 calls it "Law Student Assessor / Qualified Supervisor". Capella inherits one naming convention. Until canonical, the model can't be used as a TRR verification source.
Blocks: Capella model, TRR gate
Options: Update ARM to use "Assessment Org", or update AOM to use ARM's breakdown
3
Who holds the DBA — Calmly or pool operator?
The FMOS calculation shifts dramatically based on whether Calmly holds the DBA (combined 69.3% P50) or the pool operator holds it (defendant-only revenue model ~9.3%). This decision affects the fundraising use of the financial model.
Blocks: Financial model, investor materials
Options: Legal opinion on DBA holder. Run both FMOS scenarios in the fin sim.
