Key sensitivity. The FinSim v4 sensitivity analysis identifies defendant settlement rate (PB-03) as the dominant driver of FMOS variance across the Monte Carlo distribution. At the 70% settlement rate assumption, FMOS is 60.3% (P50). If the actual settlement rate falls below approximately 50%, the P50 FMOS drops below the 60% CDR threshold — this is a structural failure mode, not an execution one. Assessment org EV accuracy (WP2) is the second-order sensitivity: a failure there collapses both claimant activation (CP2) and defendant settlement (CP3) simultaneously, making it a correlated risk, not an independent one.

P10 interpretation. The P10 result of 55.8% reflects a plausible adverse combination of lower-than-assumed settlement rate and higher-than-assumed assessment org costs. It is below the 60% gate, but P10 is not the gate criterion — P50 is. The P10 figure is noted here because it informs the condition attached to the gate: PB-03 (defendant settlement rate) must be tested first in the build sequence, before any capital is committed to downstream flows.

Gate verdict
PASS WITH CONDITION. P50 of 60.3% clears the 60% CDR threshold by 0.3 percentage points. Condition: Part1 (ITP channel) and CP3 (defendant settlement) must be validated before the exchange platform build commits to downstream flows. PB-03 is sequenced first in the build plan. If PB-03 pilot produces a settlement rate below 60%, the P50 FMOS falls below the CDR threshold and the C2 architecture must be redesigned before implementation continues.

2

Revenue model architectural decision — PP-C BALM clarification required

The Stage 4 simulation uses a combined DBA revenue model: PP-D defendant fee + PP-C claimant DBA fee = Calmly platform revenue. This gives P50 FMOS 69.3%.

The architectural question unresolved: does Calmly hold the DBA and distribute net to the pool operator, or does the pool operator hold the DBA? If Calmly holds it, PP-C (~£600 at median) is Calmly revenue and the combined model is correct. If the pool operator holds it, PP-C is pool operator revenue only, and the defendant-only model applies — giving P50 FMOS approximately 9.3%. This is not a rounding error; it is a structural decision about the legal form of the exchange relationship.

Owner: Tom Manuel + legal / BALM review. Blocking: use of the financial model for fundraising. Must resolve before investor materials are produced. Does not block CDR close or pilot design.
